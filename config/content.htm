這正是 Webots 強大之處 —— 它不僅能「模擬機器人」，還能作為控制系統設計與動態模擬的驗證平台。
下面我會一步步教你如何利用 Webots 進行控制系統設計與動態模擬，從概念 → 模型 → 控制 → 驗證。

🧠 一、整體流程概觀
你可以把 Webots 當成是一個「虛擬實驗室」：
物理模型（Webots 模擬） ←→ 控制器（你設計的控制系統）

完整流程如下：
1️⃣ 建立物理模型（機器人或系統）
2️⃣ 分析動態特性（例如質量、摩擦、關節剛度）
3️⃣ 設計控制器（PID、LQR、MPC、神經網路等）
4️⃣ 透過 Webots Controller 實作控制算法
5️⃣ 模擬、觀察響應、調整控制參數
6️⃣ 驗證穩定性與性能（例如追蹤誤差、超越量、穩態誤差）

⚙️ 二、建立可控制的物理模型
在 Webots 中，每個機械系統或機器人都具備真實物理屬性：


每個零件（body）有質量 mass、慣性矩 inertia


關節（joint）定義了可動範圍、摩擦、彈性


馬達（motor）提供可控制的力矩或速度


感測器（sensor）可量測位置、速度、角度等訊號


💻 三、建立控制器程式（Controller）
Webots 支援多種語言：Python、C、C++、MATLAB。
以 Python 為例，你可以撰寫控制器來實現控制演算法。


這樣就能實時模擬控制力矩對系統動態的影響，觀察擺桿是否穩定。

📈 四、進行動態模擬與結果分析
Webots 提供即時數據可視化與資料輸出功能：


Display 元件：可繪圖


Supervisor 控制器：能紀錄數據


Robot.getTime()：取得模擬時間


save data：輸出 .csv 檔後用 MATLAB / Python 分析


然後可以在 Python（matplotlib）或 MATLAB 繪出時間響應曲線，分析動態行為（上升時間、超越量、穩態誤差）。

🔬 五、與控制理論結合
你可以在 Webots 裡驗證不同的控制器：
PID 控制Webots 內部實作最簡單快速、適合低階控

制狀態回授（LQR）Python / MATLAB 實作適合多變量系統

Model Predictive Control (MPC)需 Python + CasADi 或 MATLAB高精度、可處理限制條件

Adaptive / Neural 控制TensorFlow / PyTorch可模擬 AI 控制行為

Hybrid SimulationWebots + Simulink完整控制迴路設計與真實模擬整合

🧩 六、與 MATLAB/Simulink 整合（進階）
如果是做控制系統設計研究（例如自動控制、機器人學課題），可以：


在 MATLAB Simulink 中建立控制系統模型


使用 Webots MATLAB API 與模擬環境連線


由 MATLAB 傳送控制命令給 Webots


Webots 回傳感測資料給 MATLAB


即可達到 硬體在環（HIL）/軟體在環（SIL）模擬


這樣能直接在 Simulink 內進行控制器設計、調參與實驗。

🧠 七、應用範例
類型模擬對象控制目標飛行器控制四軸飛行器姿態穩定、路徑追蹤機械手臂六軸手臂末端軌跡控制、阻抗控制移動機器人差速 / 全向車避障、SLAM、軌跡跟隨動力學研究雙擺、車輛懸吊系統非線性動態分析

Webots 是什麼？有什麼特點？ 如何進行場景模擬？ 如何用 Python 進行控制系統設計？

     Webots是一個跨平台的 3D 機器人模擬器，由 Cyberbotics 維護，面向教育、研究與研發使用。它提供世界編輯器、物理引擎、豐富的機器人/感測器/致動器模型、以及多語言控制介面（C/C++、Python、Java、MATLAB 等），從快速原型到複雜多機器人系統都能處理。Webots 自 2018 年起開源（Apache 2），並有官方教學與大量範例。

------------------------------------------------------------------------------------------------------------

  • 安裝與建立專案

    • 下載並安裝 Webots（Windows/macOS/Linux）或使用 Docker image。

  • 新建 world（場景）

    • 在 IDE 裡建立新的 world，放地板、牆、光源、物件或匯入你的 3D 模型（如 OBJ/WRL）。可以用「拖放 + 屬性面板」建立場景。

  • 加入機器人與控制器

    • 從範例庫拖入機器人模型，或使用 Robot Builder 自行組裝。指定 controller（選擇 Python/C++ 等），可選 Supervisor 來控制整個世界。

  • 設定模擬參數

    • 設定 time step（模擬時間步長）、物理參數（摩擦、質量）、渲染選項，以及是否啟用實時（real-time）或加速模式。

  • 寫控制程式與上傳

    • 在 controllers 目錄下建立控制程式（例如 Python 檔），啟動模擬時 Webots 會執行該 controller。可在 IDE 看到 console 輸出與即時感測器資料。

  • 執行、除錯與紀錄

    • 按 Run 開始模擬；可暫停、單步；使用 Supervisor 捕捉/修改場景；可錄影或輸出 log/CSV 做後續分析。

  • 進階：整合 ROS / 真機回傳

    • 需要與 ROS 通訊時可啟用 Webots-ROS 接口，把模擬當成真實機器人的感測與控制端。

------------------------------------------------------------------------------------------------------------

      Webots 的 Python controller 是一個普通的 Python 腳本，使用 Webots 提供的 controller 模組（from controller import Robot 等）。控制迴圈通常以 robot.step(time_step) 為同步點，必須在迴圈中定期呼叫以讓模擬前進。

-CODE範例-

# simple_controller.py
from controller import Robot, Motor, DistanceSensor

TIME_STEP = 32 # ms

robot = Robot()

# 取得馬達與感測器
left_motor = robot.getDevice('left wheel motor')
right_motor = robot.getDevice('right wheel motor')
left_motor.setPosition(float('inf')) # velocity control
right_motor.setPosition(float('inf'))

ds = robot.getDevice('ds0')
ds.enable(TIME_STEP)

while robot.step(TIME_STEP) != -1:
dist = ds.getValue()
if dist < 800:
# 避障（簡單反向轉向）
left_motor.setVelocity(-2.0)
right_motor.setVelocity(2.0)
else:
left_motor.setVelocity(3.0)
right_motor.setVelocity(3.0)
